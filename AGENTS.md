## 📂 Общая структура проекта

Вся логика размазана по двум основным директориям: `app/` (клиентское недоразумение) и `server/` (серверный костыль). 

```
.
├── app/                  # Клиентская часть (фронтенд-мусор)
│   ├── assets/           # Стили и прочая статика
│   ├── components/       # Компоненты интерфейса
│   ├── pages/            # Роутинг страниц
│   ├── stores/           # Локальное состояние (Pinia + LocalStorage)
│   ├── types/            # Схемы типов TypeScript
│   └── utils/            # Вспомогательные хелперы
├── server/               # Серверная часть (бэкенд-заглушка)
│   ├── api/              # Эндпоинты API
│   └── utils/            # Системные промпты и утилиты
├── nuxt.config.ts        # Конфиг этого чуда
└── package.json          # Зависимости, которые ты наставил
```

## 🧠 Клиентская архитектура (Фронтенд)

Вся твоя логика на клиенте завязана на **Pinia** и **VueUse** (потому что сам ты стейт сохранять в `localStorage` не умеешь).

### 1. Состояние и Хранение (`app/stores/`)
* **[chat.ts](file:///home/iquldev/Projects/GChat/app/stores/chat.ts)**: 
  Главный мозговой центр. Хранит список чатов `chats` и выбранный чат `selectedChatId` через `useLocalStorage`.
  * Сам отправляет запросы к `/api/chat/stream` методом POST.
  * Обрабатывает чтение потока ответов (SSE) через `ReadableStream` и `TextDecoder`.
  * Содержит костыли для перезапуска упавших запросов (`retryMessage`), переименования чатов (`renameChat`) и удаления (`deleteChat`).
* **[ui.ts](file:///home/iquldev/Projects/GChat/app/stores/ui.ts)**:
  Всякий визуальный мусор: открыт ли сайдбар, открыты ли настройки, какая тема сейчас включена.

### 2. Страницы и Роутинг (`app/pages/`)
Ты разделил интерфейс на две убогие страницы:
* **[index.vue](file:///home/iquldev/Projects/GChat/app/pages/index.vue)**: Главная страница. Показывает пустой экран или приветствие с предложением создать чат.
* **[chat/[id].vue](file:///home/iquldev/Projects/GChat/app/pages/chat/%5Bid%5D.vue)**: Динамический роут для конкретного диалога. Подтягивает ID из параметров ссылки и загружает переписку.

### 3. Компоненты (`app/components/`)
Произведена реорганизация компонентов по доменам. Новые папки и ключевые компоненты:

* **sidebar/**:
  * `Sidebar.vue` — контейнер
  * `ChatList.vue`, `ChatListItem.vue` — список чатов и элемент списка
  * `SidebarActions.vue` — кнопки и утилиты сайдбара
  * `SidebarSearch.vue` — поиск/фильтрация

* **chat/**:
  * `ChatMessage.vue` — верхний компонент, собирает части сообщения

  * `MessageFooter.vue` — таймштамп, действия (retry/copy)

* **compose/**:
  * `MessageInput.vue` — поле ввода и send controls
  * `NewChatForm.vue` — форма создания нового чата
  * `ModelSelector.vue` — выбор модели
  * `PromptExamplesPanel.vue` — примеры промптов

* **ui/** (примитивы):
  * `IconButton.vue`, `Avatar.vue`, `Timestamp.vue` — маленькие переиспользуемые UI-примитивы

* **homepage/**:
  * `HomepageHero.vue` — блок приветствия / демонстрационный чат

* **settings/**:

Примечания:
- После перемещений предпочитательно использовать явные импорты в страницах (Nuxt auto-import может не обнаружить новые пути сразу).
- UI-примитивы положены в `app/components/ui/` — новые агенты должны их переиспользовать, а не создавать дубликаты.
- При обновлении событий/emit-типов избегать TSTuple в SFC generics; использовать function-style defineEmits typing.

## ⚙️ Серверная архитектура (Бэкенд)

Поскольку базы данных у тебя нет, сервер просто работает прокси-сервером для запросов к Google Gemini API.

### 1. Обработка стриминга (`server/api/chat/stream.post.ts`)
* Принимает POST-запрос с историей сообщений и выбранной моделью.
* **Валидация**: Использует библиотеку `zod` для проверки структуры входящего тела запроса (`chatSchema`).
* **Gemini SDK**: Использует официальный `@google/genai` (класс `GoogleGenAI`).
* **Стриминг**: Вызывает `ai.models.generateContentStream` с системным промптом из `server/utils/prompts.ts`. 
* Отдаёт заголовки `text/event-stream` и через `Readable` стримит куски текста обратно клиенту в реальном времени.

### 2. Безопасность и Конфигурация (`nuxt.config.ts`)
* Внедрена `nuxt-security` для предотвращения базовых атак.
* Роут `/api/**` защищён ограничителем частоты запросов (`rateLimiter`: 5 токенов в минуту). Если будешь слишком часто тыкать, получишь по рукам 429 ошибкой.
* API ключ Gemini подтягивается через `process.env.NUXT_AI_API_KEY` в `runtimeConfig`.

### 📝 Заметка для разработчика (то есть для тебя, ничтожество)
Если ты решишь сломать этот код, помни:
1. Хранилище полностью зависит от локального кэша браузера. Почистишь кэш — все чаты улетят в помойку.
2. Ключ Gemini API (`NUXT_AI_API_KEY`) должен быть прописан в `.env` файле, иначе сервер пошлёт тебя на три буквы с кодом 401.
